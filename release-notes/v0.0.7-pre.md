# GoRipper v0.0.7-pre — Output Quality, String Dedup & CLI Filter Flags

**Release date:** 2026-03-13

## Install

```bash
go install github.com/muxover/goripper/cmd/goripper@v0.0.7-pre
```

Or build from source:

```bash
git clone https://github.com/muxover/goripper.git
cd goripper
git checkout v0.0.7-pre
go build -o goripper ./cmd/goripper/
```

## Summary

Makes output genuinely useful for daily analysis work. String deduplication
eliminates repeated entries that previously appeared from both the header-pair
scan and the LEA cross-reference scan. The summary now shows a full per-type
string breakdown. The pclntab version and magic are visible in the report
header. Four new CLI flags give analysts direct control over what strings
appear in output.

## What's included

**String deduplication**

`Deduplicate()` is now the final step in the extraction pipeline. Entries with
the same `(Value, Type)` are merged: their `ReferencedBy` lists are unioned and
the lower offset is kept. The result is sorted by offset ascending. Eliminates
the ~200 duplicate entries that appeared in nullmonitor output.

**Per-type string breakdown in summary**

```
Strings:   1612 total  (7 URLs · 4 IPs · 89 paths · 3 secrets · 201 pkg-paths · 1308 plain)
```

All six type counts (`url`, `ip`, `path`, `secret`, `pkgpath`, `plain`) are
now present in both JSON (`SummaryOutput`) and text output.

**Pclntab version in report header**

```
Go Version: go1.22.1
Pclntab:    version=go1.20+  magic=0xFFFFFFF1
```

`BinaryInfo.PclntabVersion` and `BinaryInfo.PclntabMagic` are populated from
the parsed pclntab and shown in text and JSON output.

**New CLI flags**

| Flag | Commands | Effect |
|------|----------|--------|
| `--min-len N` | `strings`, `analyze` | Drop strings shorter than N bytes |
| `--no-plain` | `strings`, `analyze` | Suppress all `plain`-typed strings |
| `--min-refs N` | `strings`, `analyze` | Keep only strings with ≥ N user-code references |
| `--show-refs` | `strings`, `analyze` | Print top-3 referencing functions per string |

Examples:

```bash
# IOC extraction: only high-signal strings referenced by user code
goripper strings malware.exe --no-plain --min-refs 1 --show-refs

# Find short tokens (e.g. 4-char API keys)
goripper strings malware.exe --min-len 4 --type secret
```
