# GoRipper v0.0.1-pre — Initial Release

**Release date:** 2026-03-03

### Install

```bash
go install github.com/muxover/goripper/cmd/goripper@v0.0.1-pre
```

Or download a pre-built binary from [Releases](https://github.com/muxover/goripper/releases/tag/v0.0.1-pre).

---

## Summary

First public release of GoRipper — a Go binary intelligence framework that extracts
behavioral insight from compiled Go executables (PE and ELF, x86_64). The core analysis
pipeline is functional: function extraction, call graph, string classification, type
recovery, concurrency detection, and behavior tagging all work on PE binaries built with
Go 1.18–1.24.

This is a pre-release. Known limitations are documented below.

---

## What's included

### Core pipeline
- **Binary loading** — PE (`.exe`) and ELF format support via `debug/pe` / `debug/elf`.
- **Go version detection** — via `debug/buildinfo` with pclntab magic fallback.
- **Function extraction** — `gopclntab` parsed via `debug/gosym`, covering Go 1.2–1.24.
- **Package classification** — runtime / stdlib / user / cgo automatically separated.
- **Call graph** — direct CALL edges resolved via x86 disassembly.
- **String extraction** — `.rodata` scanner with LEA/MOV RIP-relative cross-reference.
- **String classification** — URL, IP, path, secret, Go package path, plain.
- **Type recovery** — Go `rtype` descriptor parsing for struct name and kind.
- **Concurrency detection** — goroutine, channel, and mutex pattern detection.
- **Behavior tagging** — NETWORK, CRYPTO, FILE_WRITE, FILE_READ, EXEC, REGISTRY, HTTP, DNS.
- **CFG + pseudocode** — basic block splitting and pseudocode emission (optional).

### CLI
- `goripper analyze` — full pipeline
- `goripper functions` — function list with filters
- `goripper strings` — string extraction with type filter
- `goripper callgraph` — call graph tree
- JSON output mode (`--json`)
- Filters: `--only-user`, `--no-runtime`, `--pkg`, `--type`, `--depth`, `--cfg`, `--types`

## Known limitations

- **String extraction quality is low** — raw `.rodata` scan produces 10,000+ "plain" strings. Rewrite in `v0.0.2-pre` (Go string struct-based extraction).
- **ELF untested in practice** — loader written and compiles, but not validated on real ELF binaries. Fix in `v0.0.3-pre`.
- **No test suite** — zero automated tests. Coming in `v0.0.3-pre`.
- **CFG produces garbage for non-code stubs** — `go:buildid` and linker stubs disassemble as noise. Filter in `v0.0.2-pre`.
- **Dead code in `detect.go`** — unused helpers left from an earlier implementation. Cleaned up in `v0.0.2-pre`.
- **x86_64 only** — ARM64 support planned for `v0.2.0`.
