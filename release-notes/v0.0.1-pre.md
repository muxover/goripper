## Summary

First public release of GoRipper ‚Äî a Go binary intelligence framework that extracts
behavioral insight from compiled Go executables (PE and ELF, x86_64). The core analysis
pipeline is functional: function extraction, call graph, string classification, type
recovery, concurrency detection, and behavior tagging all work on PE binaries built with
Go 1.18‚Äì1.24.

This is a pre-release. Known limitations are documented below.

---

## What's included

### Core pipeline
- **Binary loading** ‚Äî PE (`.exe`) and ELF format support via `debug/pe` / `debug/elf`.
- **Go version detection** ‚Äî via `debug/buildinfo` with pclntab magic fallback.
- **Function extraction** ‚Äî `gopclntab` parsed via `debug/gosym`, covering Go 1.2‚Äì1.24.
- **Package classification** ‚Äî runtime / stdlib / user / cgo automatically separated.
- **Call graph** ‚Äî direct CALL edges resolved via x86 disassembly.
- **String extraction** ‚Äî `.rodata` scanner with LEA/MOV RIP-relative cross-reference.
- **String classification** ‚Äî URL, IP, path, secret, Go package path, plain.
- **Type recovery** ‚Äî Go `rtype` descriptor parsing for struct name and kind.
- **Concurrency detection** ‚Äî goroutine, channel, and mutex pattern detection.
- **Behavior tagging** ‚Äî NETWORK, CRYPTO, FILE_WRITE, FILE_READ, EXEC, REGISTRY, HTTP, DNS.
- **CFG + pseudocode** ‚Äî basic block splitting and pseudocode emission (optional).

### CLI
- `goripper analyze` ‚Äî full pipeline
- `goripper functions` ‚Äî function list with filters
- `goripper strings` ‚Äî string extraction with type filter
- `goripper callgraph` ‚Äî call graph tree
- JSON output mode (`--json`)
- Filters: `--only-user`, `--no-runtime`, `--pkg`, `--type`, `--depth`, `--cfg`, `--types`

## Known limitations

- **String extraction quality is low** ‚Äî raw `.rodata` scan produces 10,000+ "plain" strings. Rewrite in `v0.0.2-pre` (Go string struct-based extraction).
- **ELF untested in practice** ‚Äî loader written and compiles, but not validated on real ELF binaries. Fix in `v0.0.3-pre`.
- **No test suite** ‚Äî zero automated tests. Coming in `v0.0.3-pre`.
- **CFG produces garbage for non-code stubs** ‚Äî `go:buildid` and linker stubs disassemble as noise. Filter in `v0.0.2-pre`.
- **Dead code in `detect.go`** ‚Äî unused helpers left from an earlier implementation. Cleaned up in `v0.0.2-pre`.
- **x86_64 only** ‚Äî ARM64 support planned for `v0.2.0`.
