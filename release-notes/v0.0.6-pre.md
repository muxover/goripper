# GoRipper v0.0.6-pre — Test Coverage, ELF Parity & Input Robustness

**Release date:** 2026-03-13

## Install

```bash
go install github.com/muxover/goripper/cmd/goripper@v0.0.6-pre
```

Or build from source:

```bash
git clone https://github.com/muxover/goripper.git
cd goripper
git checkout v0.0.6-pre
go build -o goripper ./cmd/goripper/
```

## Summary

Brings test coverage from near-zero on most packages to verified levels across all core
packages. Hardens the analysis pipeline against malformed or truncated input — it no
longer crashes, it returns partial results with warnings. Adds ELF fixture-based tests
that run on any OS, a `Makefile` for local dev, and `staticcheck` compliance.

## What's included

**Crash-safe pipeline**

Every stage after `loadBinary` is wrapped in a `safeRun` recovery. A panic in any
stage — parse, string extraction, call graph, type recovery — is caught, converted to
a warning, and the pipeline continues with whatever it extracted so far. The
`Warnings []string` field on `AnalysisResult` surfaces these to callers.

**Test coverage**

| Package                   | Before | After |
|---------------------------|--------|-------|
| `internal/binary`         | 0%     | 65%   |
| `internal/behaviors`      | 0%     | 82%   |
| `internal/concurrency`    | 0%     | 91%   |
| `internal/functions`      | 0%     | 78%   |
| `internal/output`         | 0%     | 74%   |

New test files:
- `internal/binary/binary_test.go` — 9 PE tests + 5 ELF fixture tests
- `internal/functions/functions_test.go` — classifier table + integration test
- `internal/behaviors/behaviors_test.go` — all 5 behavior tag categories
- `internal/output/output_test.go` — text, JSON, warnings, filters, call graph, types
- `internal/concurrency/concurrency_test.go` — goroutine spawn, channel send, no-concurrency
- `pkg/analyzer/analyzer_test.go` — truncated-binary and zero-byte robustness tests

**ELF parity**

A pre-built `linux/amd64` fixture (`testdata/fixture_linux_amd64`) is committed to the
repo. The ELF loader is exercised on every platform without requiring a Linux runner.
A `//go:build linux` test in `internal/binary/elf_test.go` additionally exercises the
ELF path on `ubuntu-latest` in CI.

**Makefile**

```
make build   # go build ./...
make test    # go test -race -count=1 -timeout 120s ./...
make vet     # go vet ./...
make lint    # staticcheck ./...
make check   # vet + lint + test
```

**`staticcheck` clean**

Two pre-existing issues fixed: S1011 loop-over-append in `cmd/goripper/main.go` and
U1000 unused helper in `internal/strings/extractor_test.go`.

**CI**

`paths-ignore` added to CI and audit workflows — doc-only commits no longer trigger
build or vulnerability scan runs.
