# GoRipper v0.0.2-pre — String Quality + CFG Fixes

**Release date:** 2026-03-03

### Install

```bash
go install github.com/muxover/goripper/cmd/goripper@v0.0.2-pre
```

Or download a pre-built binary from [Releases](https://github.com/muxover/goripper/releases/tag/v0.0.2-pre).

---

## Summary

This pre-release focuses on output quality. The raw `.rodata` byte scanner in v0.0.1-pre
produced 13,000+ garbage string entries for a typical Go binary, making the `strings`
subcommand nearly useless in practice. v0.0.2-pre replaces it with a Go string header
`(ptr, len)` layout scanner that reduces output to ~500–2000 high-quality strings.

Five correctness fixes ship in this release. No new features.

---

## What's fixed

### Fix 1 — String extraction rewrite

**Before:** Scanned every byte of `.rodata` for printable byte runs ≥ 4 chars.
Result: 13,000+ entries, mostly garbage (fragment bytes, alignment padding, embedded data).

**After:** Scans `.rodata` for Go string header pairs `(ptr uint64, len uint64)` at
8-byte aligned offsets. Only emits strings where `ptr` points back into `.rodata`,
`len` is in the range 6–4096, and all bytes are printable ASCII.
Result: ~500–2000 high-quality strings per binary.

### Fix 2 — Remove dead helpers from `gopclntab/detect.go`

Four unused helper functions (`readUintPtr`, `readUint32`, `readUint64`, `readNullString`)
were left over from before the `debug/gosym` migration. Removed.

### Fix 3 — CFG stub guard

`goripper analyze --cfg` previously disassembled synthesized non-code functions
(`go:buildid`, `go:cgo_*`, `_cgo_*`, `type:.*`) and emitted garbage basic blocks.

The CFG builder now skips a function if its name matches a known stub prefix **or** its
first instruction is not a valid Go function prologue (`CMP`, `SUB`, `PUSH`, or `MOV`).

### Fix 4 — `StringTypePkgPath` classification

Go import paths (e.g. `golang.org/x/crypto`, `github.com/user/repo`) were previously
classified as `plain`. A new `pkgpath` string type is introduced with a matching regex.

`goripper strings <binary> --type pkgpath` now surfaces all Go module import paths
embedded in the binary.

### Fix 5 — Signed LEA displacement

`mem.Disp` in `x86asm.Mem` is a signed `int64`. The previous code cast it directly to
`uint64`, causing backward RIP-relative references (negative displacement) to underflow
to huge addresses and never match any `.rodata` offset.

The cross-reference scanner now performs signed arithmetic and casts to `uint64` only
after the bounds check passes. Backward string references are now resolved correctly.

## Known limitations (carried forward)

- **ELF untested in practice** — loader compiles but not validated on real ELF binaries. Fix in `v0.0.3-pre`.
- **No test suite** — zero automated tests. Coming in `v0.0.3-pre`.
- **x86_64 only** — ARM64 support planned for `v0.2.0`.
