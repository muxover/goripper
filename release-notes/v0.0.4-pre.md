# GoRipper v0.0.4-pre — Garble/Obfuscation Detection & Stripped Binary Resilience

**Release date:** 2026-03-07

### Install

```bash
go install github.com/muxover/goripper/cmd/goripper@v0.0.4-pre
```

Or download a pre-built binary from [Releases](https://github.com/muxover/goripper/releases/tag/v0.0.4-pre).

---

## Summary

This release adds obfuscation and garble detection, heuristic function re-labeling for
high-score binaries, string decryptor stub detection with XOR key recovery, stripped binary
resilience via PE `.pdata` fallback, and CGo boundary mapping.

---

## What's new

### Obfuscation scoring

A new `internal/obfuscation` package computes a 0.0–1.0 score for every binary using
four independent signals:

| Signal | Weight | Detail |
|--------|--------|--------|
| Shannon entropy of function names | 35% | Garble base-62 names score ≈ 3.5+ bits/char vs. normal ≈ 2–3 |
| Known-prefix ratio | 30% | < 10% `main.` / `runtime.` / `fmt.` etc. → likely garbled |
| String density | 20% | < 0.05 strings/function → strings likely encrypted |
| Build info absent | 15% | `debug/buildinfo` failure is a strong obfuscation signal |

Score → level mapping: `none` (< 0.15) · `low` (0.15–0.34) · `medium` (0.35–0.59) · `high` (≥ 0.60)

**JSON output** (`binary_info`):
```json
"obfuscation_score": 0.72,
"obfuscation_level": "high",
"obfuscation_indicators": ["high name entropy (4.81 bits/char)", "no build info (stripped or garbled)"]
```

**Text output**:
```
Obfuscation: 0.72 [high]  (high name entropy (4.81 bits/char), no build info)
```

### Heuristic function re-labeling

When `ObfuscationScore > 0.5`, garbled functions (identified by: no package dot, contains
digits, and high character entropy) receive advisory tags based on their call targets:

| Tag | Trigger |
|-----|---------|
| `[suspected:network_connect]` | calls `net.Dial`, `net.(*Dialer)`, `syscall.Connect` |
| `[suspected:file_write]` | calls `os.Create`, `os.(*File).Write` |
| `[suspected:exec]` | calls `os/exec.(*Cmd).Run`, `syscall.Exec` |
| `[suspected:encryption]` | calls `crypto/aes.*`, `crypto/rc4.*`, `golang.org/x/crypto/*` |
| `[suspected:goroutine_spawn]` | calls `runtime.newproc` |
| `[suspected:http_client]` | calls `net/http.(*Client).Do` |
| `[suspected:registry_access]` | calls `golang.org/x/sys/windows/registry.*` |
| `[suspected:large_unknown]` | size > 4096 bytes, no recognisable calls |

Tags are clearly advisory — they appear in `function.tags` alongside normal behavior tags.

### String decryptor stub detection

Garble can encrypt string literals behind small XOR stubs. GoRipper now detects these:

- **Detection**: functions with size < 100 bytes called by ≥ 50 other functions.
- **Tagging**: `[STRING_DECRYPTOR_STUB callers=N]` appended to the function's tags.
- **XOR key recovery**: scans the stub body for `XOR AL, imm8` / `XOR r/m8, imm8` /
  `XOR r/m32, imm8` patterns. If a non-zero key is found:
  `[STRING_DECRYPTOR_STUB callers=N xor=0x3f]`

New `decryptor_stubs` array in JSON output:
```json
"decryptor_stubs": [
  {"name": "aB3xK9mP", "addr": "0x4a1234", "caller_count": 312, "xor_key": "0x3f"}
]
```

New `=== String Decryptor Stubs ===` section in text output.

### Stripped binary resilience

When gopclntab is absent or fails to parse (garbled binary, non-standard linker script):

1. A `Warning` is recorded in `AnalysisResult.Warnings` (visible at the top of text output).
2. The analyzer falls back to scanning the PE `.pdata` exception table to recover
   `(address, size)` pairs and generates synthetic names: `sub_0x4a1000`, `sub_0x4a1100`, …
3. `function_source: "synthetic"` in JSON; `[SYNTHETIC]` marker in text function list.
4. New `synthetic_functions` count in `SummaryOutput`.
5. Warning emitted when > 50% of functions are synthetic.

---

### CGo boundary mapping

Go functions that directly call CGo bridge functions are now identified:

- `SummaryOutput.CgoCallSites` — list of Go function names that cross into CGo.
- `SummaryOutput.CGOFunctions` — count of CGo bridge functions detected.
- `=== CGo Boundaries ===` section in text output (only shown when CGo is present).

```
=== CGo Boundaries (3 call sites) ===
Go memory safety ends at these transition points.
  main.callNativeLib
  main.invokePlugin
  main.loadSQLiteDB
```

## New fields in output

### `AnalysisResult`
| Field | Type | Description |
|-------|------|-------------|
| `warnings` | `[]string` | Non-fatal pipeline issues (omitted when empty) |
| `decryptor_stubs` | `[]DecryptorStubOutput` | String decryptor stubs (omitted when empty) |

### `BinaryInfo`
| Field | Type | Description |
|-------|------|-------------|
| `obfuscation_score` | `float64` | 0.0–1.0 |
| `obfuscation_level` | `string` | none/low/medium/high |
| `obfuscation_indicators` | `[]string` | Human-readable reasons |

### `FunctionOutput`
| Field | Type | Description |
|-------|------|-------------|
| `function_source` | `string` | `"pclntab"` / `"symbol_table"` / `"synthetic"` |

### `SummaryOutput`
| Field | Type | Description |
|-------|------|-------------|
| `cgo_functions` | `int` | Count of CGo bridge functions |
| `synthetic_functions` | `int` | Count of synthetic `sub_0x*` functions |
| `cgo_call_sites` | `[]string` | Go functions that call into CGo |
| `decryptor_stubs` | `int` | Count of detected decryptor stubs |

## Known limitations (carried forward)

- **CMOVNE plain blobs** — 512-byte plain blobs from the CMOVNE compiler pattern are
  not yet split. Fixed in `v0.0.5-pre`.
- **ELF not integration-tested on Linux** — run `go test ./...` on Linux for ELF coverage.
- **XOR key recovery is best-effort** — only detects single-byte XOR with direct
  immediate operand; multi-byte keys and ROL patterns are not decoded.
- **Synthetic names from .pdata only** — ELF stripped binary fallback deferred to `v0.0.6-pre`.
- **x86_64 only** — ARM64 deferred to `v0.2.0`.
